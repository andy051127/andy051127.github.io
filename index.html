왜 Riverpod?
Provider 대비 타입 안전성·테스트 용이성 우수.
StateNotifier + FutureProvider 조합으로 비동기 관리.
왜 SQLite (오프라인)?
외부 서버 비용 제로. 이전 프로젝트(LLM-wiki)에서
API 비용 문제로 중단된 경험 반영.
데이터 흐름
UI → ViewModel → Repository → SQLite
단방향 흐름으로 디버깅·테스트 용이.
